Artisan

Expert

Scripts are stored in a common repository and updated after every deployment.

Object map or repository is kept up to date.

Automation tool can accommodate multiple languages.

Automation tool provides cross platform and cross browser testing capabilities.

Basic functionality tests includes performance and security concerns.

Test conditions for Non-Functional are extracted with basic functional test conditions.

Failed performance and security tests mean code failure is not allowed to go live.

Professional

Script repository being used has powerful integration capabilities.

Automation tool has satisfactory remote support.

% Automation test coverage are calculated using Lightsabre stats from Automation team.

Performance test for each functionality being developed with use of stubs and drivers.

Use of Static Code Analysis tools by Developers to improve security of overall application.

Have a realistic mechanism to define and perform security tests.

Have sufficient expertise to perform adequate security testing.

Have predefined Security Dimensions (such as Access Control, Password Control, Logging and monitoring) covered by tests in alignment with Security Team.

Security specialists are involved in scrum of srum meetings and have global knowledge of the work of all feature teams.

Automated Performance and Security Tests in place to check application against deployment environment.

Master

All artefacts are under version management.

Test related metrics also have the required level of support.

Tools provide support for test case design and defect collection.

% Automated test coverage are calculated by collaborating with Automation team for automation tools or repository (e.g lightsabre, CHEF, etc).

Constant maintenance of Non-Functional Regression pack based on Performance and Security Registers and Audits (Internal and External).

Analytics results used to improve Performance, Security and overall User Experience of Application even after Go Live.

Have threat modelling scenarios and acceptable risk levels established up front.

Set of documents that identify and justify the security requirements of the application is in place.

Security professionals and Developers sit down together to analyse applications from an attacker's point of view.

Use of Dynamic Code analysis tools (such as WebInspect) planned and used for all code deployments
